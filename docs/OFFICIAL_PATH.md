# Official Path

Use this guide if you want a path that is closer to Apple's framework and platform learning structure.

## Step 1: Objective-C Language

Start in `foundations/`.

Recommended order:

1. `foundations/02_classes_properties_and_initializers.m`
2. `foundations/03_categories_and_lightweight_generics.m`
3. `foundations/04_protocols_and_delegation_shape.m`
4. `foundations/06_blocks_and_collection_transformations.m`
5. `foundations/05_error_patterns_and_nullability_style.m`

## Step 2: Foundation Framework

Continue with Foundation-heavy files.

Recommended order:

1. `foundations/01_nsstring_nsarray_nsdictionary.m`
2. `intermediate/01_model_mapping_and_validation.m`
3. `intermediate/03_file_persistence_and_serialization.m`

## Step 3: Cocoa Patterns And Data Flow

Move into observer and collaboration patterns.

Recommended order:

1. `advanced/02_cocoa_data_flow/01_kvc_kvo_and_model_mapping.m`
2. `advanced/03_design_patterns/01_protocols_categories_and_service_design.m`

## Step 4: Runtime And Memory

Study the deeper Objective-C execution model.

Recommended order:

1. `advanced/01_runtime_and_memory/01_runtime_and_message_dispatch.m`
2. `advanced/01_runtime_and_memory/02_memory_management_and_blocks.m`

## Step 5: Concurrency

Finish with scheduling and thread-safety concerns.

Recommended order:

1. `intermediate/02_nsoperation_queue_pipeline.m`
2. `advanced/04_concurrency/01_gcd_operations_and_thread_safety.m`

## Step 6: Project Practice

Apply everything in the CLI project.

Recommended order:

1. `projects/01_task_board_cli/main.m`
2. `projects/01_task_board_cli/README.md`
